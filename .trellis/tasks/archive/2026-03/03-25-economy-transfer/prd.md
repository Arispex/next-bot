# economy 转账功能

## Goal

在 economy 插件中新增「转账」命令，允许用户将金币转给其他已注册用户。

## 命令格式

```
转账 <用户 ID/@用户/用户名称> <数量>
```

## 校验规则

| 检查项 | 失败提示 |
|--------|---------|
| 参数数量不对 | raise_command_usage() |
| 数量非整数 | 转账失败，数量必须为整数 |
| 数量 ≤ 0 | 转账失败，数量必须大于 0 |
| 发起方未注册 | 转账失败，请先注册账号 |
| 目标用户不存在 | 转账失败，用户名称不存在 |
| 目标用户名称不唯一 | 转账失败，用户名称不唯一，请使用用户 ID 或 @用户 |
| 目标用户 ID 解析失败 | 转账失败，用户参数解析失败 |
| 转给自己 | 转账失败，不能转账给自己 |
| 目标用户 DB 不存在 | 转账失败，目标用户不存在 |
| 余额不足 | 转账失败，金币不足（当前：X） |

## 数据库操作

- 原子扣减发起方金币，增加目标方金币
- 单次 session.commit()

## Acceptance Criteria

- [ ] 命令解析：支持用户 ID / @用户 / 用户名称
- [ ] 全部校验规则通过
- [ ] 转账成功后双方金币正确更新
- [ ] 成功消息显示：转出金额、对方名称、当前余额
